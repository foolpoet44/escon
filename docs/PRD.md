# 제품 요구사항 문서 (PRD)
## ESCO Skills Ontology Explorer

---

## 1. 개요

### 1.1 제품명
**ESCO Skills Ontology Explorer** (에스코 스킬 온톨로지 익스플로러)

### 1.2 제품 비전
유럽 기술·역량·자격·직업 분류체계(ESCO)의 디지털 스킬 및 역량 데이터를 **Factory Robotics(스마트 팩토리 로봇기술)** 관점에서 시각화하고 탐색할 수 있는 플랫폼.
**Robotics Tech for Smart Factory**를 실현하기 위한 핵심 역량(Skills)과 직무(Occupations)의 관계를 규명하고, 제조 현장에 최적화된 인재 양성 및 기술 매칭을 지원합니다.

### 1.3 버전
- **현재 버전**: v0.1.0 (Phase 1 MVP)
- **문서 작성일**: 2026년 2월 7일

---

## 2. 배경 및 목적

### 2.1 문제 정의
- ESCO(European Skills, Competences, Qualifications and Occupations) 프레임워크는 방대한 양의 스킬 데이터를 포함하고 있으나, 기존 인터페이스는 사용자 친화적이지 않음
- 로봇기술(Robot Tech Stack) 관련 스킬과 역량을 체계적으로 탐색하기 어려움
- **Smart Factory** 환경에서 요구되는 로봇 공정, 제어, 센싱/인지, 통합 기술의 연관성을 파악하기 어려움

### 2.2 목표
1. **접근성 향상**: 복잡한 온톨로지 데이터를 직관적인 UI로 제공
2. **탐색 효율화**: 도메인별, 스킬 유형별 필터링 및 검색 기능 제공
3. **시각화**: 스킬 관계 및 통계를 시각적으로 표현
4. **확장성**: 향후 기능 확장을 위한 모듈화된 아키텍처 구축

## 3. 대상 사용자

### 3.1 주요 사용자
| 사용자 유형 | 설명 | 주요 니즈 |
|------------|------|----------|
| HR 전문가 | 채용 및 역량 관리 담당자 | 직무별 필요 스킬 파악, 역량 매핑 |
| 교육 설계자 | 커리큘럼 개발자 | 스킬 갭 분석, 학습 경로 설계 |
| 연구자 | AI/로보틱스 연구원 | 기술 동향 분석, 연구 방향 설정 |
| 정책 입안자 | 직업 훈련 정책 담당자 | 산업별 스킬 수요 분석 |

### 3.2 사용 시나리오
1. **스킬 탐색**: 사용자가 특정 도메인(예: AI/ML)의 스킬 목록을 브라우징
2. **검색**: 특정 키워드로 관련 스킬 검색
3. **분석**: 도메인별 스킬 분포 통계 확인
4. **시각화**: 스킬 간 관계를 그래프 형태로 탐색

## 5. 데이터 구조

### 5.1 도메인 목록 (10개)
| 도메인 키 | 한글명 | 설명 | 스킬 수 |
|----------|--------|------|---------|
| robotics | 로보틱스 | 로봇 조립, 유지보수, 인간-로봇 협업 | 72 |
| perception | 인지 | 컴퓨터 비전, 센서, 이미지 처리 | 593 |
| control | 제어 | 제어 시스템, 액추에이터, 모션 제어 | 155 |
| mechatronics | 메카트로닉스 | 전자기계 시스템, 유압, 공압 | 234 |
| ai_ml | AI/ML | 머신러닝, 신경망, 딥러닝 | 3,472 |
| manufacturing | 제조 | 산업 자동화, 생산 시스템, 품질 관리 | 561 |
| manipulation | 조작 | 그래스핑, 피킹, 엔드 이펙터 | 81 |
| navigation | 내비게이션 | 경로 계획, 위치추정, SLAM | 69 |
| safety | 안전 | 안전 시스템, 충돌 회피, 위험 평가 | 99 |
| integration | 통합 | 시스템 통합, 임베디드, PLC/SCADA | 62 |

### 5.2 스킬 타입
| 타입 | 설명 |
|------|------|
| knowledge | 지식 (Know-What) |
| skill/competence | 기술/역량 (Know-How) |

### 5.3 스킬 데이터 스키마
```typescript
interface Skill {
    uri: string;         // ESCO 공식 URI
    label: string;       // 스킬명
    type: SkillType;     // 스킬 유형
    description: string; // 상세 설명
}
```

## 7. 아키텍처

### 7.1 디렉토리 구조
```
escon/
├── app/
│   ├── page.tsx              # 홈페이지/대시보드
│   ├── layout.tsx            # 공통 레이아웃
│   ├── globals.css           # 전역 스타일
│   ├── components/           # 재사용 컴포넌트
│   │   ├── DomainCard.tsx    # 도메인 카드
│   │   ├── SkillCard.tsx     # 스킬 카드
│   │   ├── Navigation.tsx    # 네비게이션
│   │   ├── SearchBar.tsx     # 검색바
│   │   └── MermaidChart.tsx  # Mermaid 차트
│   ├── lib/                  # 유틸리티 & 데이터
│   │   ├── types.ts          # TypeScript 타입
│   │   ├── constants.ts      # 상수 정의
│   │   └── skills-data.ts    # 데이터 로딩 함수
│   ├── domains/              # 도메인 페이지
│   ├── skills/               # 스킬 탐색 페이지
│   ├── visualization/        # 시각화 페이지
│   └── analytics/            # 분석 페이지
├── public/                   # 정적 파일
├── package.json
└── tsconfig.json
```

### 7.2 컴포넌트 구조
```
[Layout]
    └── [Navigation]
    └── [Page Content]
            ├── [Dashboard] ──→ [DomainCard] × N
            ├── [Domains]   ──→ [DomainCard], [SkillCard]
            ├── [Skills]    ──→ [SearchBar], [SkillCard] × N
            └── [Analytics] ──→ [MermaidChart], [Charts]
```
